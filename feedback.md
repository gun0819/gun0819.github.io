- 발표
  - 시스템 구축을 하게 된 동기 표현, 배경 설명 필요, 시나리오 만들기
  - 관리 주체가 누구인지 명확하게 표현
  - 독서 시스템의 필요성, 목적성 어필할 필요 (ex. 독서율 같은 통계 데이터)
  - 전제 조건을 도서관에서 읽는다고 가정하고 구현
  - 국가 예산이 필요(전주시에서 사용할 경우) -> 전주 도서관들의 데이터를 찾아볼 것(대출, 반납 여부 연동)
  - 퀴즈, 보상을 하나하나 직접 보여주면서 발표할 필요는 없다.
  - 기업과 커퓨니케이션한 내용들을 녹여낼 것
  - 판독 진위 여부를 어떻게 구현했는지 녹여낼 것
  - 개인 정보와 같이 민감한 주제들 같은 경우는 미리 말해서 질문을 차단
 

- 구현
  - 판독 진위 여부를 어떻게 할 지
  - 관리자가 판독 기준을 눈으로 확인할 수 있도록 팝업 기능 추가
  - 독서를 장려할 수 있는 다른 기능도 구현
  - 기능적으로는 충분하지만 메인 UI를 다듬을 것(통합검색 이용)
