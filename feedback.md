- 발표
  - 시스템 구축을 하게 된 동기 표현, 배경 설명 필요, 시나리오 만들기
  - 관리 주체가 누구인지 명확하게 표현
  - 국가 예산이 필요(전주시에서 사용할 경우) -> 전주 도서관들의 데이터를 찾아볼 것(대출, 반납 여부 연동)
  - 퀴즈, 보상을 하나하나 직접 보여주면서 발표할 필요는 없다.
  - 기업과 커퓨니케이션한 내용들을 녹여낼 것
 

- 구현
  - 판독 진위 여부를 어떻게
  - 독서를 장려할 수 있는 다른 기능도 구현
  - 기능적으로는 충분하지만 메인 UI를 다듬을 것(통합검색 이용)
  - 


발표시에 배경 설명이 필요하다 -> 시스템 구축을 하게된 동기를 확실하게 표현할 것 -> 도서관 관련 데이터를 찾아서 시스템의 필요성을 pt에서 다룰것

관리 주체가 누구인지 명확하게 표현할것

독서 시스템의 필요성, 목적성을 어필할 필요가 있다 -> 독서율같은 통계 등의 데이터

전제 조건을 도서관에서 읽는다는 가정 하에 구현하는게 좋다

구현 기능 활용시에 생각해볼것 -> 사용자가 팝업되는 판독 기준을 눈으로 확인할 수 있도록

판독 진위 여부를 어떻게 판독하게 되었는지 핵심을 pt에 쓰면 좋다

발표 시에 개인정보 관련해서 어떻게 대처할 지를 미리 말해주면 좋다 -> 질문 차단 가능 => OAuth2 기반 보안 도입
